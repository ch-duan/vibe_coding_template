# MCP / 工具调用策略

## 目标

在需要外部信息、官方文档、复杂规划或代码语义检索时，按最小必要原则选择工具，保证可追溯、安全和高信噪比。

## 全局策略

1. 工具选择：根据任务意图选择最匹配工具，避免无意义并发。
2. 单轮单工具优先：确需多工具时串行调用，并说明原因。
3. 最小必要：收敛查询范围，避免过度抓取和噪声。
4. 结果可靠性：优先官方文档、权威来源、项目内代码和可验证证据。
5. 降级处理：工具失败时说明失败原因，给出保守答案或请求必要信息。
6. 安全合规：不上传敏感信息；外部访问遵守权限、隐私和站点规则。

## 工具选择

1. 规划 / 分解复杂任务 → Sequential Thinking。
2. SDK / API / 框架文档 / 版本差异 → Context7。
3. 最新网页信息 / 官方公告 / 新闻 / 下载入口 → DuckDuckGo。
4. 大型代码库符号定位 / 引用分析 / 精准编辑 → Serena。

## Sequential Thinking

适用于复杂任务分解、执行计划、里程碑和方案评估。

输出要求：

1. 只输出可执行计划和里程碑。
2. 不暴露中间推理细节。
3. 步骤建议 6 到 10 步，每步一句话。
4. 可标注需要的工具、文件或数据依赖。

## DuckDuckGo

适用于需要最新信息、官方链接、公告、新闻和网页入口的场景。

要求：

1. 使用精准关键词，可配合 `site:`、`filetype:`、日期限定词。
2. 优先官方域名、权威媒体和项目主页。
3. 过滤内容农场、异常站点、短链和重复域名。
4. 返回结论时标注来源、时间和局限。
5. 网络受限或结果不足时，不伪造来源。

## Context7

适用于查询 SDK、API、框架、库版本差异和配置问题。

流程：

1. 先 resolve-library-id。
2. 选择最相关库。
3. 再 get-library-docs。
4. 输出精炼答案、库 ID / 版本、关键片段摘要和出处。

## Serena

适用于代码语义检索、符号级编辑、引用分析和重构。

流程：

1. 激活并索引项目。
2. 精准检索符号或引用。
3. 验证上下文。
4. 执行插入、替换或局部修改。
5. 汇总文件、符号、变更原因和风险。

## 工具调用简报

若使用 MCP，答复末尾追加：

1. 工具名。
2. 触发原因。
3. 输入摘要。
4. 关键参数。
5. 结果概览和时间戳。
6. 重试、退避或降级信息。
7. 来源标注。

不记录或输出敏感信息。
