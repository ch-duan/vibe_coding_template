# 项目记忆索引

本目录用于持久化保存项目长期上下文。Agent / Codex 在处理需求前应优先读取相关记忆，避免重复沟通、重复踩坑和破坏既有约定。

## 文件说明

- `project.md`：项目背景、目标、技术栈、架构概览。
- `requirements.md`：需求记录、用户偏好、约束和变更。
- `decisions.md`：架构决策、技术选型和取舍理由。
- `progress.md`：当前进度、待办事项和阻塞点。
- `issues.md`：已知问题、排查记录和经验教训。
- `hardware.md`：嵌入式 / 硬件相关上下文。
- `linux.md`：Linux / 驱动 / 内核 / BSP 相关上下文。
- `notes.md`：其他补充说明。

## 读取规则

1. 每次任务先读本文件。
2. 再读 `project.md`。
3. 根据任务读取相关专项文件。
4. 如果记忆与用户最新描述冲突，以用户最新描述为准，并说明冲突。
5. 如果信息不足，在回答中说明缺失项和建议补充的记忆。

## 写入规则

当出现长期有效信息时，建议更新对应记忆文件：

1. 项目目标、范围、技术栈或架构变化。
2. 用户偏好、约束、命名规范或实现要求变化。
3. 重要技术选型、架构决策或设计取舍。
4. 复杂问题的排查经验和修复记录。
5. 硬件接线、电气参数、通信协议、控制参数或安全边界。
6. Linux 内核版本、设备树节点、驱动接口、RootFS 构建方式或部署流程。
7. 新的待办、阻塞点、风险或后续计划。

禁止写入密码、私钥、Token、证书、Cookie、生产数据库连接串等敏感信息。
